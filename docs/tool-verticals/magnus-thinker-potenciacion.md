# Magnus Thinker - Potenciacion De Skill Pills

Fecha: 2026-05-24

## Objetivo

Ordenar Magnus como motor always-on de pensamiento y separar sus pills de conocimiento para que Codex, Claude Code y Antigravity entiendan rapido que activar en cada situacion.

Magnus no es una herramienta puntual. Es una capa cognitiva siempre activa:

- cruza variables;
- lee memoria cuando hay proyecto activo;
- detecta riesgos, oportunidades y recursos faltantes;
- activa pills segun el problema;
- corre F1-F13 internamente;
- muestra cadena visual o Coda solo cuando el usuario lo pide o cuando aporta aprendizaje real.

## Contrato Operativo

| Capa | Funcion | Visible por defecto |
| --- | --- | --- |
| Context Pull | Leer Second Brain y threads relevantes | No |
| Criterios activos | Aplicar criterios guardados y kernel general | No |
| F1-F13 | Cadena cognitiva completa | No |
| Skill Pills | Lentes especializados por tipo de problema | No |
| Advertencias/oportunidades | Riesgos, cruces, rutas, variables faltantes | Si, cuando aportan valor |
| Cadena visual | Reconstruccion pedagogica de la cadena | Solo si el usuario la pide |
| Coda Magnus | Resumen metacognitivo para afinar criterio | Si se pide o al cierre profundo |
