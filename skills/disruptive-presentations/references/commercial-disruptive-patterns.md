# Commercial Disruptive Slide Patterns

Use this reference to improve prompts for commercial, executive, sales, pitch, strategy, educational and persuasive slides.

This is not a separate output format. It is a pattern library for image-slide prompts. The final artifact remains:

- full-image slide generated by `disruptive-presentations`;
- prompt/handoff per slide;
- HTML player;
- optional PPTX export only if requested.

## Core Rule

A commercial slide should not only look premium. It should make one business argument visible in under 5 seconds.

Every slide must choose one dominant persuasive mechanism:

- contrast;
- tension;
- proof;
- simplification;
- inevitability;
- risk;
- opportunity;
- transformation;
- decision;
- momentum.

## Pattern Picker

| Pattern | Use when | Visual structure | Prompt move |
| --- | --- | --- | --- |
| Before / After | Showing transformation, value proposition, improvement | Split world, left degraded/right improved | "Show a clear visual contrast between current pain and desired future state." |
| Tension Gap | Audience feels stuck between current reality and desired future | Two poles with a visible gap/bridge | "Make the gap visible as the core obstacle." |
| Problem Stack | Many small frictions accumulate into one big issue | Vertical stack, pressure, overload | "Show compounding friction, not isolated issues." |
| Bottleneck | One constraint blocks the whole system | Narrow choke point in a flow | "Make the constraint visually dominate the system." |
| Wedge | A small strategic entry expands into market/control | Thin entry widening forward | "Show a narrow initial move becoming a larger advantage." |
| Flywheel | Repeated actions compound into momentum | Circular loop with accelerating weight | "Show compounding loops, not a one-time funnel." |
| Funnel With Friction | Leads/users/options reduce through stages | Funnel with leakage points | "Show where value is lost and what fixes the leak." |
| Ladder / Maturity | Moving from basic to advanced capability | Tiered ascent, levels, milestones | "Show progression as status and capability gain." |
| Control Tower | One system orchestrates many moving parts | Central command node with monitored signals | "Show visibility, coordination and decision control." |
| Operating System | Many tools become one repeatable workflow | Layered OS, modules, inputs/outputs | "Show repeatable structure replacing scattered effort." |
| Iceberg | Visible symptom hides deeper drivers | Surface layer plus hidden depth | "Show the hidden cause bigger than the visible problem." |
| Bridge | A method connects two disconnected states | Span over gap, stable supports | "Show method as the credible path across uncertainty." |
| Lens / Filter | A framework reveals what was invisible | Lens, spotlight, before/after perception | "Show the framework changing what can be seen." |
| Map / Territory | Strategy requires orientation | Map, route, landmarks, risks | "Show decision-making as navigation." |
| Trade-off Matrix | Audience must choose between options | 2x2 or axis map with highlighted zone | "Make the preferred position inevitable." |
| Proof Wall | Evidence must build credibility | Grid/wall of evidence, one claim highlighted | "Show many signals converging into one conclusion." |
| Domino Chain | One decision triggers downstream effects | Sequential falling or activated blocks | "Show consequence and sequence." |
| Signal vs Noise | Clarify what matters in clutter | Bright signal path through noise field | "Show focus emerging from chaos." |
| Cost Of Inaction | Doing nothing is visibly expensive | Decay line, accumulating debt, leaking tank | "Show inaction as an active loss." |
| Asymmetric Bet | Small input can create large upside | Small lever moving large object | "Show leverage, not effort." |
| Trust Gap | Buyer/client hesitates due to risk | Barrier between promise and proof | "Show proof assets closing the trust gap." |
| Decision Gate | A threshold must be crossed | Gate, checkpoint, green/red paths | "Show criteria deciding the next action." |
| Ecosystem Hub | One platform connects many actors | Hub-and-spoke with differentiated actors | "Show connection and role clarity, not a generic network." |
| Timeline Compression | AI/automation reduces long cycle | Long path compressed into short path | "Show time saved as spatial compression." |

## Commercial Slide Families

### Sales / Proposal

Use:

- Before / After;
- Trust Gap;
- Proof Wall;
- Cost Of Inaction;
- Decision Gate;
- Bridge;
- Wedge.

Avoid:

- generic handshake photos;
- icon rows for benefits;
- fake dashboards;
- vague "growth" arrows without mechanism.

### Strategy / Executive

Use:

- Tension Gap;
- Trade-off Matrix;
- Map / Territory;
- Wedge;
- Asymmetric Bet;
- Flywheel;
- Control Tower.

Avoid:

- overdecorated strategy pyramids;
- 12-node ecosystems with no reading path;
- KPI clutter without a decision.

### Product / SaaS / Automation

Use:

- Operating System;
- Bottleneck;
- Timeline Compression;
- Control Tower;
- Signal vs Noise;
- Ecosystem Hub.

Avoid:

- generic SaaS dashboard screenshots;
- floating UI cards that do not prove anything;
- automation as decorative glowing lines.

### Education / Thesis / Training

Use:

- Lens / Filter;
- Iceberg;
- Ladder / Maturity;
- Bridge;
- Domino Chain;
- Map / Territory.

Avoid:

- textbook bullet lists;
- decorative metaphors unrelated to the concept;
- overloaded conceptual diagrams.

## Prompt Template

When applying a pattern, inject this block before the final image prompt:

```text
Commercial pattern: [pattern name].
Persuasive mechanism: [contrast/tension/proof/etc.].
Business argument: [one sentence].
Visual structure: [split/gap/stack/funnel/lens/map/etc.].
What must be obvious in 5 seconds: [viewer takeaway].
Visible text: [exact title + anchor phrase + labels].
Avoid: generic corporate deck, icon grid, stock dashboard, vague arrows, decorative tech background.
```

## QA Gate

Reject or regenerate if:

- the slide is beautiful but the business argument is unclear;
- the pattern is visible but does not match the message;
- the visual relies on generic icons instead of a mechanism;
- the slide has more than one main argument;
- the viewer cannot explain what changed, what matters, or what to do next.

## Source Notes

Patterns are synthesized from:

- Duarte presentation contrast/sparkline and visual storytelling principles;
- business/data storytelling principles: one clear takeaway, context before data, guide the audience to a decision;
- common executive/pitch deck structures;
- ESC-AI disruptive slide production learnings.
