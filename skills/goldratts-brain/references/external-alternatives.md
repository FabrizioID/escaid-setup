# External Alternatives For Operational Systems

Use before designing a custom operating process when a real-world alternative may already exist.

## Search Targets

- Existing SaaS or no-code tools.
- BPMN/DMN patterns and examples.
- Workflow automation templates.
- SOP/playbook templates.
- GitHub repos and awesome lists.
- Reddit/operator discussions about real tool failures.
- Industry benchmarks and case studies.

## Evaluation Criteria

| Criterion | Question |
| --- | --- |
| Fit | Does it match the user's workflow and team maturity? |
| Adoption | Will non-technical people actually use it? |
| Bottleneck relief | Does it remove the real constraint or only digitize it? |
| Visibility | Does it make status, owner and next action visible? |
| Automation readiness | Is the process stable enough to automate? |
| Cost/friction | Is setup faster than custom design? |
| Lock-in | Does it trap the team in a tool too early? |

## Decision

Present alternatives as:

1. Use existing tool/template.
2. Adapt external pattern.
3. Build custom process.
4. Automate only after the process is stable.

Do not recommend automation before the human process is clear.
