# Table Editing

## Reuse Existing Good Tables

If the document already contains a validated table with the right look and spacing, reuse that pattern. Copying structure is usually safer than inventing a new table.

## Safe Procedure

1. Locate the table start index or table identifier.
2. Confirm which rows are headers, section bands, data rows, totals, notes, or spacer rows.
3. Confirm which cells are merged.
4. Insert or update row content first.
5. Merge cells only after the target row exists.
6. Apply background color after merging.
7. Re-read the table structure after row insertion, deletion, merge, or style changes.

## Table Meaning

Before editing a table, identify what each row sells, explains, proves, or groups. For commercial documents, combine with `technical-quotation-builder` so proposal rows, deliverables, activities, optional items, and notes do not get mixed.

For academic tables, combine with APA/thesis skills when captions, notes, numbering, or source logic matter.

## Avoid

- rebuilding a stable table from scratch unless necessary
- applying broad formatting changes across unrelated rows
- assuming the same index after a prior structural edit
- merging cells before confirming the row exists
- leaving full body cells in bold unless the template already does so
