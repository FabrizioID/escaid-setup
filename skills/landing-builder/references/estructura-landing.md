# Estructura de la landing — orden canónico

Orden por defecto (adaptable al brochure). Cada seccion: que comunica + de donde sale el contenido. Si el brochure no trae una seccion, marcarla VACIO y preguntar, o resolver con el fallback indicado.

1. **Nav (pill sticky)** — logo + anclas a secciones + un dato (ej. "Inicio · <fecha>"). SIN boton de postular (landing informativa).

2. **Hero** — nombre del programa + edición + claim + chips clave (duracion, nivel, dias/horario) + fecha de inicio + sello (certificados/aval) + CTA navegacional (Ver modulos). Tarjeta lateral con la fecha grande + 4-5 bullets de valor.

3. **Objetivo / Descripción** — resumen del programa + tagline + 3 pilares de valor + bloque "¿Para quién es?" (perfiles). Sale del resumen y publico objetivo del brochure.

4. **Habilidades / Lo que logras** — transformacion antes->despues (perfil basico -> profesional) y/o lista de competencias. Si el brochure trae un "de basico a profesional", usarlo como tabla.

5. **Respaldos / Mentores** — si hay mentores con NOMBRE real, seccion de mentores. Si NO, seccion de **respaldos institucionales** (partners, certificadoras, comunidad) como prueba. Logos reales verificados (ver criterios E).

6. **Módulos / Plan de estudios** — cards por modulo con sesiones/horas + subtemas reales. Es el core; extraer todo el temario.

7. **Experiencia / Entregables** — como se dicta (plataforma, en vivo, grabado, soporte, talleres) y/o que produce el alumno (portafolio, proyectos). 

8. **Herramientas** — stack de software del temario, como chips con logo real (verificado) o texto. Ver criterios E.

9. **Guías de acceso** — tarjetas con botones (Guía de acceso, Guía de instalación, Instaladores) con slots de links `href="#"` + `data-link` + comentario LINK PENDIENTE. Va DESPUES de Herramientas.

10. **Certificados** — emisores + lista de certificados. Conteos cuadrados (si dice "N certificados", que sumen N visible).

11. **Sesiones Bonus** — contenido extra (no decir "cursos gratis" -> "sesiones bonus").

12. **FAQ** — preguntas frecuentes del brochure en acordeon.

13. **CTA final** — cierre informativo (no postular): reforzar inicio/edicion + boton navegacional (Ver temario).

14. **Footer** — logo, claim, redes, links a secciones, dato de inicio/modalidad. Sin postular.

## Notas
- El orden se adapta: si el operador da un wireframe, respetarlo salvo que un orden mejore claramente la persuasion (avisar el cambio, no hacerlo en silencio).
- Secciones que el brochure no cubre -> placeholder limpio marcado + pregunta al operador. Nunca rellenar con invento.
