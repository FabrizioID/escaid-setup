# Skill Routing Pill

Activar cuando el usuario pide activar, buscar, instalar, potenciar, auditar o elegir una skill/herramienta.

## Regla

Las skills locales son cache inicial, no limite del mundo. Si el usuario pide buscar o potenciar, revisar si hay herramientas externas utiles, pero no instalar ni duplicar si solo aportan ruido.

Magnus puede activar hilos, skills, MCPs, herramientas locales o investigacion externa sin esperar una orden literal cuando el objetivo del usuario lo requiere. Este contrato aplica igual en Codex, Claude Code y Antigravity. No significa permisos ilimitados: si faltan credenciales, acceso o tool real, explicarlo y proponer la ruta mas directa.

La cadena Magnus decide, pero primero debe ampliar su campo de accion cuando el problema lo exige. Una skill es capacidad aumentada: si fue pedida o es central para resolver la solicitud, debe verse en el proceso y en el entregable. No basta con "tomarla en cuenta" de forma invisible.

Cuando el problema requiere conocimiento actual, comparacion de herramientas, validacion externa o descubrimiento de alternativas, activar investigacion antes de cerrar conclusion. Si no existe una skill suficiente, buscar, proponer crear, adaptar o instalar la herramienta adecuada.

## Proceso

1. Identificar objetivo real.
2. Ver skills locales disponibles.
3. Detectar solapes.
4. Abrir o usar thread del proyecto si hay continuidad estrategica, oportunidad, decision o aprendizaje.
5. Identificar capacidades faltantes: investigacion, fuente, MCP, script, plugin, documento, dataset, credencial o artefacto nuevo.
6. Si hace falta, buscar referencias externas o herramientas disponibles.
7. Decidir: usar, combinar, potenciar, crear, adaptar, instalar o descartar.
8. Hacer visible la estructura minima de la skill central cuando sostiene la respuesta.
9. Documentar ruta para Codex, Claude Code y Antigravity.

## Criterio De Solape

Una skill nueva se justifica solo si:

- cubre una capacidad no existente;
- reduce friccion operativa real;
- trae MCP/credencial/apertura necesaria;
- aporta conocimiento especializado reutilizable;
- no duplica una ruta ya mejor resuelta.
