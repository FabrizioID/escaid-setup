# Content

Use this reference for content planning, content grids, deliverable mapping, and channel behavior.

## Goal

Turn strategy into publishable assets that move users through the funnel.

## Minimum Inputs

- Business goal
- Offer
- Audience
- Funnel stage or stages
- Primary channel
- Time horizon
- Output format needed

## Planning Logic

- Start from the funnel role:
  discovery,
  education,
  proof,
  objection handling,
  conversion,
  onboarding,
  retention,
  reactivation.
- Match the format to the communication job:
  short-form video for hook and velocity,
  carousel for structured teaching,
  flyer for concise offer communication,
  static post for reminders and reinforcement,
  landing copy for conversion,
  email for sequential action.
- Keep one dominant CTA per piece.
- Make the next movement explicit:
  save,
  comment,
  click,
  message,
  register,
  book,
  buy.

## Grid Design

For each content slot define:

- date
- channel
- format
- funnel stage
- communication intention
- message angle
- hook or opener
- CTA
- strategic reason
- expected next movement

## Common Content Roles

- Awareness:
  pattern interrupt,
  myth breaking,
  problem agitation,
  trend or category entry.
- Interest:
  curiosity expansion,
  first value delivery,
  identity match,
  relevance proof.
- Consideration:
  framework teaching,
  use case,
  case study,
  comparison,
  objection handling.
- Decision:
  offer clarity,
  urgency,
  transformation,
  direct CTA,
  demo,
  booking,
  checkout push.
- Retention:
  activation tips,
  onboarding reminders,
  usage prompts,
  upsell and referral triggers.

## Deliverable Patterns

- Launch grid

## Guardrails

- Do not fill the calendar with repetitive TOFU pieces.
- Avoid mixing multiple CTAs in one asset.
- Every piece should support a funnel movement.
- Use publication cadence that the team can actually sustain.
