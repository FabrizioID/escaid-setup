# Diagram Families for Miro Work

Use this file to choose the right map before drawing.

## Core Principles

1. Think in bands, not isolated boxes.
2. Use color as semantic role, not decoration.
3. Centralize one key decision or focal node when possible.
4. Prefer grouped systems over giant timelines.
5. Use layout to show hierarchy and importance.
6. Keep labels short.
7. Show loops intentionally.
8. Build modular families of diagrams instead of one overloaded canvas.

## Family 1. Growth Map

Use for:

- acquisition
- activation
- conversion
- nurturing
- retention
- reactivation

Recommended structure:

1. title bar
2. acquisition band
3. activation band
4. decision node
5. split into converts vs does not convert
6. retention band
7. return loop

## Family 2. Systemic Map

Use for:

- architectures
- operating models
- ecosystems
- data systems
- service layers

Recommended structure:

1. title bar
2. top input or external layer
3. central core or orchestration layer
4. lower execution or consumption layer
5. side commentary only if it adds meaning

## Family 3. Persona Board

Use for:

- user understanding
- objections
- motivation
- JTBD
- empathy maps

Recommended structure:

1. title + demographic strip
2. central empathy board
3. supporting panels:
   interests, pains, frustrations, goals

## Family 4. Onboarding Flow

Use for:

- product journeys
- lifecycle stages
- signup to activation flows
- guided operational sequences

Recommended structure:

1. title bar
2. stage headers
3. grouped micro-blocks under each stage
4. primary path highlighted
5. alternate paths with lighter emphasis

## Family 5. BPMN Clean

Use for:

- role-based processes
- handoffs
- step accountability
- operations with decisions and parallel routes

Recommended structure:

1. lanes or implied responsibility zones
2. short task labels
3. decision nodes only where necessary
4. minimal edge text
5. keep the route compact and auditable

## Color Logic

- dark neutral: headers and dividers
- blue: inputs, capture, top-of-funnel, source systems
- purple: orchestration, logic, strategy, intelligence
- green: conversion, retention, positive outputs
- orange/yellow: paid ads, pressure, nurture, summarized output
- gray: bridges, neutral systems, helper blocks

## Default Workflow

1. choose family
2. draft premium `HTML/PDF`
3. validate clarity and hierarchy
4. adapt to Miro
5. split into multiple board modules if needed
