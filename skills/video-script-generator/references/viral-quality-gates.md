# Viral Quality Gates

Use these gates to prevent correct-but-flat scripts.

This reference distills the uploaded review checklist and Formula 100K material into operational rules. Do not quote the source documents; apply the rules.

## Idea Qualification

Before committing to a script angle, check whether the idea has at least 6 of these traits:

- A 10-year-old could understand the core point.
- It can create controversy, disagreement, or comments.
- It is inspired by an observable viral reference or known winning format.
- The viewer would perceive the content as valuable enough to pay a small amount for it.
- At least half of a broad audience could identify with it or recognize someone close to them in it.
- It avoids unnecessary technical jargon.
- It gives a clear path from point A to point B.
- The promise is realistic and actionable for the viewer.
- If borrowed from a common idea, it adds a clear original twist.

If fewer than 6 traits are present, strengthen the angle before writing hooks.

## Purpose Selection

Every video should primarily serve one of three purposes:

- Viralidad: reach, comments, shares, attention, emotional spread.
- Valor: trust, authority, teaching, problem-solving.
- Venta: interest, desire, proof, conversion.

For warm audiences that already know the course, default to `Valor + Venta`: teach a real difference first, then show why the course matters.

## A-B Reward Rule

The hook must name or imply a reward path:

- From confusion to clarity.
- From generic understanding to specialist understanding.
- From hidden risk to visible criterion.
- From common mistake to better decision.
- From "I know this" to "I was missing this."

If the reward is not visible in the hook, rewrite it.

## 3-Second Hook Stack

The first 3 seconds should include all three:

- Verbal hook: what is said.
- Visual hook: what interrupts the feed without audio.
- Textual hook: 5-7 words max, distinct from subtitles.

The textual hook should highlight one strong word with color, size, or motion.

## Verbal Hook Formula

Use:

```text
clarity + context + curiosity
```

Or:

```text
impact + problem/promise + intrigue
```

Operational rules:

- 5 seconds or less.
- Short and direct.
- Start with a strong word when useful: Error, Cuidado, Nunca, Secreto, Mentira, Riesgo, Nadie, 3, Esto.
- Add a conflict or belief break.
- Avoid vague claims like "BIM hospitalario es importante."

## Hook Styles To Add

Use these in addition to the base taxonomy:

- Negative warning: "No hagas X..."
- Prohibition: "Nunca uses X si..."
- Information gap: "Seguro no sabias esto sobre..."
- Challenge: "El 90% falla aqui..."
- Rule of 3: "3 errores / 3 capas / 3 senales..."
- Rule of 2: direct comparison of two options.
- Controversial claim: strong opinion with proof.
- Limiting belief: "Si crees X, te equivocas."
- Statistic hook: credible number or proportion.
- Psychology hook: viewer behavior or decision bias.
- Filtering hook: directly names the audience.
- Impact summary: strong result first, explanation later.

## Visual Hook Rules

The visual hook must work without audio.

Use one or more:

- sudden camera movement or zoom
- unusual angle
- physical object or model that represents the idea
- fast cut between simple vs complex
- visible clash, red mark, warning, or failure
- hand gesture pointing to the key text
- before/after or wrong/right split screen

Avoid visual clutter. Use cuts in moderation unless editing style is the differentiator.

## Textual Hook Rules

- 5-7 words max.
- Different typographic style from normal subtitles.
- One keyword emphasized.
- High contrast.
- Appears synchronized with the spoken hook.
- Should be understandable at first glance.

Examples of text patterns:

- "NO es solo modelar"
- "El error invisible"
- "Colegio vs hospital"
- "4 capas criticas"
- "BIM con riesgo real"

## Value Delivery

Every script must deliver at least one of:

- clear actionable steps
- a digestible metaphor for a complex idea
- a concrete example
- a broken-down success or failure case
- a checklist the viewer can use immediately

For technical topics, translate jargon into concrete consequences.

## Authority Rule

Show authority through:

- proof
- a case
- a real criterion
- a specific decision
- a technical consequence
- borrowed authority when direct authority is not available

Do not rely on "I know this" tone.

## CTA Rule

The CTA must produce an action:

- comment a keyword
- save the video
- share with a specific type of person
- ask for a module/resource
- visit the profile/course page

Prefer imperative CTA, not passive invitation.

## Editing And Scene Checklist

When production direction is included, specify:

- at least 3 plane or scene changes
- no dead air, redundancies, or long silences
- minimal subtitles, centered, readable
- key words visually emphasized
- no distracting overlays, stickers, or excessive sound effects
- cuts every 3-4 seconds if the shot is static
- format should be understandable from the first second

## Stronger Structures From Formula 100K

Use these when the standard frameworks feel too polite:

### Negative Viral

Negative hook -> opportunity -> actionable steps -> promise -> save/share CTA.

### Problem / Hero

Problem -> create villain -> hero solution -> simple steps -> profile or lead-magnet CTA.

### Reaction To Comment

Show comment/video -> point out what is wrong -> practical solution -> ask for more questions.

### False Declaration

Limiting belief -> opportunity -> proof/case -> actionable refutation -> CTA.

### Invisible Problem

Disruptive doubt -> hidden problem -> symptoms/evidence -> easy solution -> CTA.

### Rule Of 3

Desire or pain with measurable goal -> proof -> 3 actionable tips -> promise -> keyword CTA.

### Paradox

Contradictory statement -> personal/context setup -> revelation -> explanation -> debate CTA.

### Mirror

Say what the audience feels but has not verbalized -> validate -> explain cause -> reframe -> comment CTA.

### Boomerang

Show result first -> failed attempt -> worse failed attempt -> final reveal -> identification CTA.

### Contrarian Challenge

Provocative claim -> why the common idea is wrong -> proof/case -> new perspective -> debate CTA.
